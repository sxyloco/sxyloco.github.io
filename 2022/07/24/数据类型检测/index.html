数据类型检测

typeof

typeof[value]：用来检测数据类型的运算符

return 返回值

  • 首先是个字符串
  • 字符串中包含对应的数据类型，例如：’number’ 、’object’、’undefined’、’symbol’、’function’

局限性

  • typeof null => “object” 二进制 null–>000 对象类型—> 000
  • 不能具体区分对象数据类型的值（无法检测是正则还是数组等）

优势

使用方便，特别是检测基本类型值（除null），还有函数类型值

instanceof

本意是用来检测实例是否隶属于某个类的运算，基于这种方式，也可做某些数据类型检测，如数组、正则…

@局限性

不能处理基本类型的值

只要再当前实例的原型链（__ proto ）中出现过的类，检测结果都是true （用户可能手动修改原型链的指向：example. proto __或者类的继承中 等情况）

constructor

原理：在类的原型上一般都会带有constructor属性，存储当前类本身，利用这一点，验证是否为所属类，从而进行数据类型检测

@局限性：constructor属性值太容易被修改

Object.prototype.toString.call([value])

调用Object原型上的toString方法，让方法执行的时候，方法中的this是要检测的数据类型，从而 获取到数据类型所属类的详细

@信息模块

“[object 所属类]”，例如：”[object Array]”…

在所有的数据类型中，他们原型上都有toString方法，除object.prototype.toString不是把数据值转换为字符串，其余的都是转换为字符串，而object原型上的toString是检测当前实例隶属类的详细信息的（检测数据类型）

obj.toString()

  • 首先基于原型链查找机制，找到object.prototype.toString
  • 把找到的方法执行，方法中的this–> obj
  • 方法内部把this（obj）的所属类信息输出 =>方法执行，方法中的this是谁，就是检测书的所属类信息

检测数据类型 方法封装

老版JQ采用的形式

let _obj = {
    isString:'String',
    isBoolean:'Boolean',
    isNumeric:'Number',
    isNull:'Null',
    isUndefined:'Undefined',
    isPlantObject:'Object',
    isSymbol:'Symbol',
    isArray:'Array',
    isDate:'Date',
    isMath:'Math',
    isRegExp:'RegExp',
    isWindow:'Window'
  },
  _toString = _obj.toString,
  _type = {};
  for (const key in _obj) {
    if (!_obj.hasOwnProperty(key)) break
    let reg = new RegExp('\\[object '+_obj[key]+'\\]')
    _type[key] = function(value){
      return reg.test(_toString.call(value))
    }
  }
  console.log(_type.isNumeric(0))

加号除了数字运算还有字符串拼接

Number : NaN /isNaN/infinity/parseInt/Number()

把其他类型转换为数字的方法

  • Number()它是按照浏览器从底层机制，把其他数据类型转化为数字
    • 字符串：看是否包含非有效数字，包含非有效数字结果就是NaN;’’=>0;
      • 引申: 遵循按照字符串从左到右查找的机制找有效数字字符(所以传递的值一定是字符串，不是也要转换为字符串然后再查找) 与Number()函数运行的底层机制并不一样
      • parseInt()=>字符串非有效数字之前的数字整数部分，parseFloat()=> 字符串非有效数字(除第一个小数点之外)之前的数字部分
      parseInt('a23123') => NaN
      parseInt('23123a') => 23123
      parseInt('undefined') => parseInt('undefined') => NaN
      parseInt('') => NaN // 因为没有找到有效数字
    • 布尔：true -> 1; false->0
    • null: -> 0
    • undefined: -> NaN
    • 引用数据类型先转换为字符串在转换为数字
      • {}/正则/函数等 -> NaN
      • [] -> ‘’ -> 0
      • [‘12’] -> ‘12’ -> 12
      • [‘21’,’23’] -> ‘21,23’ NaN
    • isNaN 遇到不是数字的会调Number()的转换机制
  • 强转化（基于底层机制转换的）Number([value])
    • isNaN(‘12px’)

      规则 ：在使用 isNaN进行检测的时候，首先会验证检测的值是否为数字类型，如果不是，先基于Number()这个方法，把值转换为数字类型，然后再检测

      isNaN([val]) [val]中的[]: 参数描述占位符

      1. 数字类型值进行数学运算只要有NaN参与,执行结果都是NaN；
      2. 如果是加法运算 , 有一个是字符串，那么，执行字符串拼接操作；
    • 数学运算 
      let res = '3' + 2
    • 两个等号（==）
      • 对象==字符串 对象转换为字符串
      • null==undefined 但是和其他值都不想等
      • 剩下两边不同都转换为数字
  • 弱转换（基于一些额外的方法转换）parseInt([value])/parseFloat([value])
    • parseInt([value]

      字符串非有效数字之前的数字整数部分

    • parseFloat([value])

      字符串非有效数字(除第一个小数点之外)之前的数字部分
